# 需求文档

根据搜索结果，现有的调度模拟器大多是教学演示工具，功能相对基础。我们需要突破传统教学模拟的局限，对接真实场景。

## 设计要求

1、课程设计时间为1周；
2、开发语言不限（C++\JAVA\C#\ASP.net…）；
3、设计文档书写规范，程序界面友好、健壮性强。

## 题目

进程控制和进程调度算法的模拟实现

### 1、设计目的

进程是操作系统最重要的概念之一，进程控制和进程调度是操作系统重要的核心内容。通过对进程控制过程的模拟实现，掌握进程的创建、阻塞、唤醒、撤销等进程控制方法的实现过程，深入理解进程在执行过程中如何通过进程控制来完成进程各状态间的转换，从而达到多进程高效率并发执行和协调、实现资源共享的目的。而在进程调度中，调度算法性能的优劣将直接影响操作系统的性能。通过对进程调度算法的模拟设计，在掌握各种调度算法的同时，进一步加深理解处理机管理技术的原理及特点。

### 2、设计内容

- （1）模拟实现进程控制过程。设计一个有多个进程并发的进程调度程序。当系统进入运行，在完成必要的初始化工作以后便进入进程调度，首先选择优先级最高的进程使其进入执行（分配CPU）。当执行进程因等待某个事件被阻塞或唤醒某个等待进程时，转入进程调度。分别按基于优先级的时间片轮转调度算法和多级反馈队列轮转调度算法对进程进行模拟调度，进程的优先级静态设置。每个进程都处于执行E(execute)、就绪R(ready)、等待W(wait)和完成F（Finish）状态之一，并假定初始状态均为R。如果被唤醒的进程的优先级大于现行的执行进程，则剥夺现行进程的执行权，而将CPU分配给被唤醒的进程。当系统处于死锁或所有进程都执行完毕时系统退出运行。
- （2）进程调度算法：基于优先级的时间片轮转调度算法和多级反馈队列轮转调度算法。基于优先级的时间片轮转调度算法可任意选择进程数量，各进程到达时间、运行时间和时间片大小，其中时间片大小、进程到达时间（从0开始）和运行时间均为整数。进程优先级初值为1~100之间的整数，数值越小优先级越高。进程每执行完一个时间片优先级下降（如优先级+2或3）。多级反馈队列轮转调度算法设置队列数为3，每级队列时间片大小按队列编号大小依次增加来选择。时间片大小、进程数量、进程到达时间和运行时间要求与基于优先级的时间片轮转调度算法相同。要求设计出相应算法，模拟出算法中各进程的执行过程并能动态显示，即每次进行进程调度时，各进程执行情况（执行了多少时间，还剩多少时间没有执行，进程的状态，各队列中进程排队的情况），列出各进程的开始执行时间、结束时间、周转时间，计算平均周转时间并对两种调度算法进行性能分析比较。注：每个进程有一个进程控制块（PCB）表示。进程控制块可以包含如下信息：进程名、优先级、到达时间、需要运行时间、已用CPU时间、进程状态等等。进程的运行时间以时间片为单位进行计算。
- （3）模拟实现过程要求使用可视化图形界面进行操作和显示（鼓励采用动画等实时动态方式显示算法执行过程）。
